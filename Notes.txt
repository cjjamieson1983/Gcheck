SWAP SPACE ISSUE

  ❗ What you’re seeing

  From your screenshot:

  Free Swap Space: 🔴 0 B
  Everything else (disk, temp, response time) = ✅ OK

  👉 So the issue is:

  Your Jenkins node has no swap space available

  🧠 What that actually means

  On your Jenkins node (your EC2 instance):

  Swap = backup memory (RAM overflow)
  When RAM fills up → system uses swap
  You currently have:
  ❌ No swap configured OR already fully used

  ⚠️ Why Jenkins is warning you

  Jenkins monitors node health and flags:

  Low disk ❌
  High load ❌
  No swap ❌

  👉 Even if things are working now, this can cause:

  Builds failing randomly
  Jenkins freezing
  Plugins crashing
  Terraform runs dying mid-execution



You need to edit the User Data script to create a swap file on boot

  ✅ Where to put it

Add the swap setup AFTER package installs but BEFORE Jenkins starts

👉 In your script, place it right after this section:

dnf install -y java-21-amazon-corretto-devel fontconfig terraform jenkins

# =========================
# Create swap file (2GB)
# =========================
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

# Make swap persistent
echo '/swapfile swap swap defaults 0 0' >> /etc/fstab
